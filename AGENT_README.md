# Агент Гарант - Юрист

## Описание

ИИ-юрист для подготовки юридических документов на основе правовой системы Гарант.

## Возможности

- Поиск и анализ действующего законодательства
- Изучение судебной практики (Сутяжник)
- Подбор DOCX-шаблонов по запросу
- Подготовка юридических документов в формате DOCX
- Автоматическая расстановка гиперссылок на нормы права
- Управление кейсами с понятной структурой папок
- Экономия API-лимитов Гарант

## Использование

### Запуск агента через opencode

Убедитесь, что конфигурация `.opencode/` скопирована в корень рабочей папки:

```powershell
# Из папки D:\ГАРАНТ\
cd garant-mcp
xcopy /E /I .opencode ..\.opencode

# Запуск агента из D:\ГАРАНТ\
cd D:\ГАРАНТ
opencode --agent garant-lawyer
```

### Альтернативный запуск MCP-сервера

```powershell
cd garant-mcp
.\venv\Scripts\Activate.ps1
python -m garant_mcp.server
```

### Рабочий процесс

1. **Создание кейса** — агент создаёт папку `кейсы/Описание_кейса_YYYY-MM-DD/`
2. **Исследование** — поиск законодательства и практики через `search_and_cite_tool`
3. **Уточнение** — вопросы пользователю для сбора фактов
4. **Подготовка DOCX** — подбор шаблона, заполнение placeholder'ов
5. **Проверка и выдача** — финальная проверка, сохранение в `результат/`

### Структура папок кейса

```
кейсы/
└── Описание_кейса_YYYY-MM-DD/
    ├── исходные данные/          # Входные документы и описание запроса
    ├── результат/                 # Готовые документы для клиента (DOCX)
    └── служебное для агента/      # Исследования и промежуточные результаты
```

## DOCX-шаблоны

Доступные шаблоны в `results/templates/`:

- `Претензия.docx`
- `Исковое_заявление.docx`
- `Договор.docx`
- `Соглашение.docx`
- `Правовое_заключение.docx`
- `Жалоба.docx`
- `Ответ_на_претензию.docx`

## Экономия лимитов API

- Используйте `search_and_cite_tool` для поиска норм и формирования ссылок.
- `export_document_*`, `download_image_tool`, `download_formula_tool` расходуют платный лимит экспорта (30/мес).
- Экспорт используйте только по явному запросу пользователя.

## Пример использования

```python
# Создать кейс
await create_case_tool("Возврат автомобиля дилеру")

# Найти законодательство
laws = await search_and_cite_tool("защита прав потребителей автомобиль")

# Изучить практику
practice = await search_judicial_practice_tool("возврат автомобиля гарантийный ремонт")

# Подобрать шаблон
template = await find_template_by_request_tool_wrapper("претензия на возврат автомобиля")

# Скопировать шаблон в кейс
await copy_template_to_case_tool_wrapper("Претензия.docx", case_path, "Претензия_возврат_авто.docx")

# Заполнить placeholder'ы
await render_docx_template_tool_wrapper(
    "Претензия.docx",
    f"{case_path}/результат/Претензия_возврат_авто.docx",
    json.dumps({
        "ФИО_истец": "Иванов Иван Иванович",
        "адрес_истец": "г. Москва, ул. Примерная, д. 1",
        "наименование_адресат": "ООО \"Автосалон\"",
        ...
    })
)
```

## Безопасность

- Никогда не фантазирует и не придумывает нормы
- Использует только проверенные источники из Гарант
- Задает уточняющие вопросы перед созданием документов
- Сохраняет промежуточные результаты
- Предупреждает о рисках

## Конфигурация

Настройки агента в `docs/agent-config.json` и `.opencode/opencode.json`.

## Требования

- Python 3.11+
- Доступ к API Гарант
- Установленные зависимости из requirements.txt
